## Context

采风阶段已跑通（s07），过堂是造物流的第二个 Agent 阶段。需要 3 个 Agent（说客、诤臣、掌秤）严格串行协作。

## Goals / Non-Goals

**Goals:**

- 说客和诤臣严格串行（说客先说，诤臣针对说客反驳）
- 辩论内容有实质（引用数据、有来有回）
- 掌秤裁决合理（通过/否决/有条件）
- 局势条计算合理
- 局中人页面可用

**Non-Goals:**

- 不追求辩论的完美质量（持续精调 prompt）
- 不实现过堂回放动画增强（属于打磨阶段）

## Decisions

### D1: 每次更鼓推进一整轮

按流水线设计.md：每次 tick 推进一整轮（说客+诤臣各说一次），不是单个 Agent 的单次发言。掌秤在最后一次 tick 单独裁决。

### D2: 局势条用 LLM 评估

每轮辩论结束后调用 LLM 评估双方论点强度，输出 shuikeScore / zhengchenScore / reason。不用规则引擎。

### D3: 最多 4 轮 + 裁决

3 轮必须，第 4 轮可选（分歧仍大时）。掌秤裁决独立一次 tick。

## Risks / Trade-offs

- **辩论质量**：高度依赖 prompt。→ 模拟 3-5 场完整过堂迭代精调。
- **串行延迟**：每轮需要两次 Agent 调用。→ 过堂阶段更鼓间隔设为 10 分钟。
