# 国际化（i18n）

栈：`i18next` + `react-i18next`，初始化在 `app/i18n/index.ts`，翻译文件在 `app/i18n/locales/{zh-CN,en}.json`。

## 支持的语言

| 代码 | 用途 |
|---|---|
| `zh-CN` | 默认，简体中文 |
| `en` | 英文 |

**当前只支持这两种**。要加新语言时，在 `app/i18n/locales/` 下加文件并在 `index.ts` 注册，同时更新 `app/stores/settings.ts` 的 `lang` 类型。

## 什么要翻译，什么不翻译

### 必须翻译

- 用户可见的 UI 文本：标题、按钮、标签、说明、错误消息、toast
- 菜单项、tooltip、placeholder、aria-label

### 禁止翻译

- 业务标识 / 代码：service 名、event type、status 码（`OK` / `DEGRADED`）
- 版本号、构建号、时间戳、日志格式
- 文件路径、URL、IP、ID
- 品牌名、产品代号（`MERIDIAN`）

> 原则：**中文环境下保持英文反而更专业**的东西一律不翻译。技术术语走英文、面向用户的措辞走翻译。

## 翻译键命名

**层级结构**：`<模块>.<分组>.<字段>`，点号分隔，小写 camelCase。

```jsonc
// ✅ 好
{
  "deck": {
    "hero": { "label": "系统状态", "description": "..." },
    "metrics": { "title": "核心指标", "uptime": "运行时长" },
    "controls": { "theme": "切换主题", "language": "切换语言" }
  },
  "settings": {
    "title": "设置",
    "theme": { "light": "浅色", "dark": "深色", "system": "跟随系统" }
  }
}

// ❌ 坏
{
  "hero_label": "...",           // 扁平下划线，层级丢失
  "DECK_HERO_LABEL": "...",      // 全大写，不是常量
  "deckHeroLabel": "..."         // 一整坨，没分组
}
```

**铁律**：

1. **按模块分组**：页面 / 组件对应顶层键（`deck` / `settings` / `about`）
2. **同一模块的键必须在 `zh-CN.json` 和 `en.json` 同位置出现**，缺一个就是 bug
3. **禁止键里混拼音**（`zhuti` → `theme`）
4. **禁止 key 当 fallback**：`t("xxx.yyy")` 里的 key 未定义时 i18next 会把 key 显示出来，这是提醒你没加翻译，不是"默认值"
5. 新加功能时**两份 locale 文件必须同步更新**，禁止只加一种语言

## 使用模式

### 组件内读取

```tsx
import { useTranslation } from "react-i18next"

function Header() {
  const { t } = useTranslation()
  return <h1>{t("deck.hero.label")}</h1>
}
```

### 动态键（按数据取翻译）

```tsx
// ✅ 用字面量模板，键里的变量部分要在 locale 文件里穷举
{STATS.map((stat) => <span>{t(`deck.metrics.${stat.key}`)}</span>)}

// 对应 locale：
// { "deck": { "metrics": { "uptime": "...", "requests": "..." } } }
```

### 切换语言

**唯一入口**：`app/stores/settings.ts` 的 `setLang`。

```tsx
const setLang = useSettingsStore((s) => s.setLang)
const { i18n } = useTranslation()

const next = "en"
i18n.changeLanguage(next)   // 切 i18next 运行时
setLang(next)               // 持久化到 zustand
```

**禁止**：只改 `i18n.changeLanguage` 不改 store——下次启动会还原。也禁止只改 store 不调 `changeLanguage`——当前页面不会刷新。

### 非组件场景读翻译

极少数情况需要在组件外拿翻译（比如主进程发过来的 toast），用 `i18n.t()`：

```ts
import i18n from "@/i18n"
toast.error(i18n.t("updater.failed"))
```

## 占位符与复数

**带变量**：用 `{{name}}` 占位。

```jsonc
{ "greeting": "欢迎回来，{{name}}" }
```

```tsx
t("greeting", { name: user.name })
```

**复数**：用 i18next 的 `_one` / `_other` 后缀（英文），中文不区分可以只写一份。

```jsonc
{
  "items_one": "{{count}} item",
  "items_other": "{{count}} items",
  "items_zh": "{{count}} 项"
}
```

实际很少遇到，遇到时再查 i18next 文档，不要自己拼字符串。

## 反模式

| 反模式 | 原因 |
|---|---|
| 组件里写死中文字符串 `<span>设置</span>` | 面向用户的文本必须走 `t()` |
| `t("some.key") || "fallback"` | 缺 key 应该报错而不是静默 fallback |
| 把业务数据（service 名、版本号）塞进 locale | 那不是翻译对象 |
| `t("settings." + dynamicVar)` 动态拼接未穷举的 key | 会导致键丢失检测不到 |
| 在 `zh-CN.json` 加了新键不同步 `en.json` | 切英文直接显示键名 |
| 翻译文本里写 HTML 标签 | 用 `<Trans>` 组件或拆成多个 key |
| 用 i18n key 做状态标识（`t("status.ok")` 当判断依据） | key 是 UI 文本，不是枚举值 |
