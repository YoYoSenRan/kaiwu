# 实施路线图

## 原则

- **只做展示网站**：当前阶段聚焦 apps/site，不涉及 console 管理后台
- **垂直切片**：每个里程碑交付一个可运行的端到端能力，而不是逐层堆砌
- **先窄后宽**：先让一个关键词完整走完流水线，再扩展到批量处理
- **流水线和网站同步推进**：流水线产出数据，网站消费数据，两者相互驱动

---

## Milestone 0 — 地基

**目标**：搭建数据层和展示网站骨架。

- [ ] 数据库 schema 设计与迁移（packages/db）
- [ ] 核心类型定义（packages/domain）
- [ ] OpenClaw 集成封装（packages/openclaw）
- [ ] 事件系统基础实现（EventBus）
- [ ] 展示网站初始化（apps/site 基础路由、布局、视觉设计系统）

**交付物**：空的但可运行的展示网站 + 数据库就绪

---

## Milestone 1 — 探测（第一个 Agent 跑起来）

**目标**：在展示网站上输入关键词 → Scout Agent 产出探测报告 → 结果实时展示。

- [ ] Scout Agent 实现（Web 搜索 + 竞品分析 + 评分）
- [ ] 网站首页：Hero 区 + 关键词输入框 + "此刻正在发生"
- [ ] 网站关键词广场页：排行 + 实时探测进度
- [ ] 探测报告的结构化输出与展示
- [ ] SSE 实时推送（Agent 状态变更 → 网站更新）

**交付物**：可以在网站上提交关键词并实时看到探测报告

---

## Milestone 2 — 论证（核心看点）

**目标**：探测通过的关键词进入多 Agent 辩论 → 辩论过程在网站上可实时旁听和回放。

- [ ] Optimist / Skeptic / Pragmatist 三个 Agent 实现
- [ ] 辩论引擎：多轮对话调度、发言规则约束、局势分析
- [ ] 辩论记录持久化（debates 表）
- [ ] 网站辩论直播：实时显示正在进行的辩论，局势进度条
- [ ] 网站辩论回放：历史辩论的完整回放体验
- [ ] 网站 Agent 面板：角色卡片、状态、关系图谱
- [ ] 务实派自动裁决 + 墓地自动归档

**交付物**：完整的全自动"探测 → 论证 → 裁决"链路，辩论是网站上最吸引人的内容

---

## Milestone 3 — 故事（产品诞生记）

**目标**：积累了足够的项目数据后，构建完整的产品故事体验。

- [ ] 产品故事列表页（Netflix 风格卡片墙）
- [ ] 产品诞生记详情页（滚动式沉浸叙事，章节结构）
- [ ] 被否决项目的墓志铭页面
- [ ] 关键词墓地（安息之地）
- [ ] Agent 详情页：成长档案、名场面集锦、战绩
- [ ] Agent 语录墙

**交付物**：网站有了丰富的、可消费的故事内容

---

## Milestone 4 — 构建 + 审查（后半段流水线）

**目标**：Architect → Builder → Inspector 跑通，产品故事可以讲到"建造"和"检验"。

- [ ] Architect Agent 实现（PRD + 技术架构 + 任务拆解）
- [ ] Builder Agent 实现（代码生成，半自动模式）
- [ ] Inspector Agent 实现（代码审查 + 安全扫描）
- [ ] 网站项目看板（生产车间监控大屏）
- [ ] 产品故事补充"蓝图"、"建造"、"检验"章节
- [ ] 漏斗数据可视化

**交付物**：产品故事可以讲 5 个阶段

---

## Milestone 5 — 发布（闭环）

**目标**：Deployer Agent 部署 → 产品上线 → 故事完结。

- [ ] Deployer Agent 实现（对接 Vercel）
- [ ] 自动部署流程 + 冒烟测试
- [ ] 产品橱窗（已上线产品展示，可直接访问）
- [ ] 产品故事的"面世"终章 + 致谢页
- [ ] 产品上线时的庆典效果
- [ ] 名人堂（从关键词到产品的成功案例）

**交付物**：第一个关键词完整走完 6 阶段，产出可访问的线上产品

---

## Milestone 6 — 打磨

**目标**：从"能用"到"想一直看"。

- [ ] 多项目并行流水线
- [ ] Agent 情绪系统 + 成长等级
- [ ] 辩论回放的动画增强（局势条动画、气泡动画）
- [ ] 产品故事的社交分享（截图卡片 + OG 标签）
- [ ] "幕后花絮"页面（技术解密）
- [ ] 被否决项目的"复活"机制（提交改进版关键词）
- [ ] 项目订阅通知（"结果出来时通知我"）
- [ ] 性能优化（SSE 连接管理、ISR、数据库查询）

**交付物**：可对外展示的完整产品

---

## 风险与应对

| 风险                                  | 影响         | 应对                                              |
| ------------------------------------- | ------------ | ------------------------------------------------- |
| OpenClaw Agent 能力不足以生成可用代码 | M4-M5 延期   | M4 先做半自动模式，人工补充代码                   |
| 探测报告质量不稳定                    | 用户体验差   | 设计 prompt 模板 + 人工标注反馈循环               |
| 辩论内容空洞无趣                      | 核心卖点失效 | 精心设计 Agent 人设 + 辩论规则 + 引入真实数据约束 |
| 部署自动化复杂度高                    | M5 延期      | 先只支持 Vercel 一种目标                          |
| 网站容易陷入无限打磨                  | 进度失控     | 每个 Milestone 有明确的"完成定义"，到了就停       |
| 初期没有真实数据，网站显得空          | 第一印象差   | M1-M2 阶段用种子关键词跑出一批真实数据再对外开放  |
