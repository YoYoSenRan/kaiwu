# 用户参与体系

## 核心理念

访客不只是观众，也是参与者。他们提交的关键词、投出的票，直接影响流水线的走向。

---

## 认证

### 登录方式

- **首发**：GitHub OAuth（受众匹配，防刷可靠）
- **后续扩展**：Google OAuth（降低非技术用户门槛）

### 触发时机

浏览网站不需要登录。以下操作需要登录：

- 提交关键词
- 投票
- 订阅项目进度

---

## 关键词提交

### 规则

- 每人每天限提交 **1 个关键词**
- 必须包含两部分：
  - **关键词**：1~20 字
  - **一句话理由**："为什么觉得值得做"，20~200 字
- 理由会展示在物帖墙上，本身就是内容

### 提交后的流程

```
用户提交
  │
  ▼
进入关键词池（status: pending）
  │
  ▼
展示在物帖墙，接受投票
  │
  ▼
按权重排序，排到队首时进入流水线
  │
  ▼
无论结果如何，提交者收到通知
```

### 重复关键词处理

- 提交时检查是否已存在相同或高度相似的关键词
- 如果已存在：提示用户"这个关键词已有人提交，你可以直接投票支持它"
- 如果已进入流水线或已完成：提示用户"这个关键词已经有故事了"并给出链接

---

## 投票机制

### 双向投票

每个关键词有两个投票按钮：

- **🟢 值得做**——"我觉得这个有市场"
- **🔴 不看好**——"我觉得这个悬"

不是简单的赞/踩，而是呼应说客和诤臣的对抗关系。

### 投票规则

- 每人每个关键词只能投一次（🟢 或 🔴 二选一）
- 投完可以改票，但不能撤票
- 每人可以投多个不同关键词，无每日上限

### 投票数据的使用

- 展示在关键词卡片上：`🟢 47 · 🔴 23`
- 进入论证阶段时，说客和诤臣可以引用这个民意数据
  - 说客："70% 的投票者看好这个方向"
  - 诤臣："但有 23 个人不看好，让我看看他们担心什么"

---

## 权重排序

关键词池中的排序决定了哪个关键词优先进入流水线。

### 权重公式

```
权重 = (green_votes × 3) + (red_votes × 2) + time_decay_bonus + submitter_bonus
```

| 因子       | 计算方式               | 权重 | 设计意图                    |
| ---------- | ---------------------- | ---- | --------------------------- |
| 🟢 票数    | green_votes × 3        | 高   | 越多人看好，越值得探索      |
| 🔴 票数    | red_votes × 2          | 中   | 争议大 = 辩论精彩，也要优先 |
| 时间衰减   | 提交越久分越低         | 中   | 保持关键词池的新鲜感        |
| 提交者加成 | GitHub 星数 > 100 加分 | 低   | 有行业经验的人的判断略加权  |

**核心设计：🔴 票也是正向加分**。一个 47🟢 23🔴 的关键词比一个 50🟢 0🔴 的更值得辩论——争议本身就是看点。

### 进入流水线的触发条件

- 权重排到队首 + 下一次心跳到来 → 自动进入探测阶段
- 同时进入流水线的项目数上限：由心跳配置中的并发数控制

## 提交者身份

### 创意发起人

关键词如果进入流水线（无论最终成功还是失败），提交者获得：

- 造物志中的 **"创意发起人"署名**
- 故事开头："由 @username 提交于 2026-03-16"
- 如果最终产品上线，致谢页永久展示

## 通知

### 触发条件

| 事件             | 通知谁          | 内容                                               |
| ---------------- | --------------- | -------------------------------------------------- |
| 关键词进入流水线 | 提交者          | "你提交的「极简记账」开始探测了"                   |
| 论证完成         | 提交者 + 投票者 | "「极简记账」通过了论证" 或 "「社交健身」被否决了" |
| 产品上线         | 提交者 + 投票者 | "「极简记账」上线了！你是这个产品的创意发起人"     |
| 项目封存         | 提交者          | "「社交健身」封存了。看看完整的过堂过程 →"         |

### 通知方式

- **站内通知**（首发）：登录后在导航栏显示未读数
- **邮件通知**（后续）：关键节点发邮件（需用户 opt-in）

---

## 防滥用

| 风险           | 措施                                       |
| -------------- | ------------------------------------------ |
| 刷票           | GitHub OAuth 实名 + 每人每关键词限一票     |
| 垃圾关键词     | 每天限 1 个 + 必须写理由 + 后台可标记 spam |
| 恶意关键词     | 内容过滤（敏感词 + AI 审核）+ 举报机制     |
| 机器人批量注册 | GitHub 账号年龄 > 30 天才能提交            |
