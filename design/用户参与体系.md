# 用户参与体系

## 核心理念

访客不只是观众，也是参与者。他们提交的关键词、投出的票，直接影响流水线的走向。

---

## 认证

### 登录方式

- **首发**：GitHub OAuth（受众匹配，防刷可靠）
- **后续扩展**：Google OAuth（降低非技术用户门槛）

### 触发时机

浏览网站不需要登录。以下操作需要登录：

- 提交关键词
- 投票
- 订阅项目进度

---

## 关键词提交

### 规则

- 每人每天限提交 **1 个关键词**
- 必须包含两部分：
  - **关键词**：1~20 字
  - **一句话理由**："为什么觉得值得做"，20~200 字
- 理由会展示在物帖墙上，本身就是内容

### 提交后的流程

```
用户提交
  │
  ▼
进入关键词池（status: pending）
  │
  ▼
展示在物帖墙，接受投票
  │
  ▼
按权重排序，排到队首时进入流水线
  │
  ▼
无论结果如何，提交者收到通知
```

### 重复关键词处理

- 提交时检查是否已存在相同或高度相似的关键词
- 如果已存在：提示用户"这个关键词已有人提交，你可以直接投票支持它"
- 如果已进入流水线或已完成：提示用户"这个关键词已经有故事了"并给出链接

## 投票机制

### 双向投票

每个关键词有两个投票按钮：

- **🔴 盖印**——按上朱印，表示"我觉得这个值得造"（朱砂红）
- **⚪ 留白**——不盖印，留白以示存疑（灰白）

不是简单的赞/踩，而是呼应说客和诤臣的对抗关系。

### 投票规则

- 每人每个关键词只能投一次（盖印 或 留白 二选一）
- 投完可以改票，但不能撤票
- 每人可以投多个不同关键词，无每日上限

### 投票数据的使用

- 展示在关键词卡片上：`🔴 47 · ⚪ 23`
- 进入论证阶段时，说客和诤臣可以引用这个民意数据
  - 说客："70% 的投票者盖了印，看好这个方向"
  - 诤臣："但有 23 个人留了白，让我看看他们担心什么"

## 权重排序

物帖池中的排序决定了哪个物帖优先进入造物流。

### 权重公式

```
权重 = (seal_votes × 3) + (blank_votes × 2) + time_decay_bonus + submitter_bonus
```

| 因子       | 计算方式 | 设计意图 |
| ---------- | -------- | -------- |
| 🔴 盖印数  | `seal_votes × 3` | 越多人盖印，越值得探索 |
| ⚪ 留白数  | `blank_votes × 2` | 争议大 = 过堂精彩，也要优先 |
| 时间衰减   | `max(0, 10 × (1 - days_since_submit / 30))` | 30 天内线性衰减，30 天后归零 |
| 提交者加成 | `github_stars >= 100 ? 5 : github_stars >= 10 ? 2 : 0` | 有行业经验的人的判断略加权 |

**核心设计：留白票也是正向加分**。一个 47🔴 23⚪ 的物帖比一个 50🔴 0⚪ 的更值得过堂——争议本身就是看点。

**示例**：

```
物帖 A：30🟢 10🔴，提交 3 天，提交者 150 stars
权重 = (30×3) + (10×2) + max(0, 10×(1-3/30)) + 5
     = 90 + 20 + 9 + 5 = 124

物帖 B：50🟢 0🔴，提交 20 天，提交者 5 stars
权重 = (50×3) + (0×2) + max(0, 10×(1-20/30)) + 0
     = 150 + 0 + 3.3 + 0 = 153.3

物帖 C：10🟢 8🔴，提交 1 天，提交者 200 stars
权重 = (10×3) + (8×2) + max(0, 10×(1-1/30)) + 5
     = 30 + 16 + 9.7 + 5 = 60.7
```

### 进入造物流的触发条件

- 当前无造物令在跑 + 物帖池非空 → 取权重最高的物帖进入采风
- 已进入造物流的物帖从池中移除（status 改为 `in_pipeline`），不再参与排序
- 已被封存或已开物的物帖也从池中移除

## 提交者身份

### 创意发起人

关键词如果进入流水线（无论最终成功还是失败），提交者获得：

- 造物志中的 **"创意发起人"署名**
- 故事开头："由 @username 提交于 2026-03-16"
- 如果最终产品上线，致谢页永久展示

## 通知

### 触发条件

| 事件             | 通知谁          | 内容                                               |
| ---------------- | --------------- | -------------------------------------------------- |
| 关键词进入流水线 | 提交者          | "你提交的「极简记账」开始探测了"                   |
| 论证完成         | 提交者 + 投票者 | "「极简记账」通过了论证" 或 "「社交健身」被否决了" |
| 产品上线         | 提交者 + 投票者 | "「极简记账」上线了！你是这个产品的创意发起人"     |
| 项目封存         | 提交者          | "「社交健身」封存了。看看完整的过堂过程 →"         |

### 通知方式

- **站内通知**（首发）：登录后在导航栏显示未读数
- **邮件通知**（后续）：关键节点发邮件（需用户 opt-in）

---

## 防滥用

| 风险           | 措施                                       |
| -------------- | ------------------------------------------ |
| 刷票           | GitHub OAuth 实名 + 每人每关键词限一票     |
| 垃圾关键词     | 每天限 1 个 + 必须写理由 + 后台可标记 spam |
| 恶意关键词     | 内容过滤（敏感词 + AI 审核）+ 举报机制     |
| 机器人批量注册 | GitHub 账号年龄 > 30 天才能提交            |
