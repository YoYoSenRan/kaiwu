## 关键技术决策

### 造物流引擎

**方案**：自建轻量状态机 + 更鼓驱动，不引入重型工作流引擎

理由：

- 阶段数固定（6 个），不需要动态编排
- 需要深度定制每个阶段的行为（过堂辩论、试剑回炉等）
- OpenClaw 本身提供了 Agent 调度能力，引擎只需管状态流转和更鼓节奏
- 全自动运行，无人工干预

核心接口：

```ts
interface PipelineEngine {
  /** 启动新造物令 */
  start(keywordId: string): Promise<Project>

  /** 更鼓响起，推进所有进行中的造物令一步 */
  tick(): Promise<TickResult[]>

  /** 获取造物令当前状态 */
  getStatus(projectId: string): Promise<ProjectStatus>

  /** 重试失败的阶段 */
  retry(projectId: string, phaseId: string): Promise<void>
}

interface TickResult {
  projectId: string
  action: "advanced" | "completed" | "failed" | "waiting"
  detail: string
}
```

### 更鼓（心跳驱动）

**方案**：全局定时器 + OpenClaw HEARTBEAT.md

整条造物流由一个全局更鼓驱动。每次更鼓响起，编排层遍历所有进行中的造物令，推进一步。OpenClaw Gateway 的心跳机制负责唤醒对应的 Agent。

```ts
interface HeartbeatConfig {
  /** 全局更鼓间隔（毫秒） */
  interval: number

  /** 各阶段可覆盖全局间隔 */
  phaseOverrides?: {
    scout?: number
    council?: number    // 辩论轮次之间，节奏更快
    architect?: number
    builder?: number    // 锻造任务之间，节奏更慢
    inspector?: number
    deployer?: number
  }
}
```

### 实时推送

**方案**：Server-Sent Events (SSE)

理由：

- 数据流向单一（服务端 → 客户端），不需要 WebSocket 的双向通信
- 实现简单，Next.js Route Handler 原生支持
- 浏览器原生自动重连
- Agent 状态变更、过堂发言、阶段流转通过 SSE 推送到展示网站

```ts
// apps/site/src/app/api/events/stream/route.ts
export async function GET(): Promise<Response> {
  const stream = new ReadableStream({
    start(controller) {
      const listener = (event: PipelineEvent) => {
        controller.enqueue(`data: ${JSON.stringify(event)}\n\n`)
      }
      eventBus.subscribe(listener)
    },
  })

  return new Response(stream, { headers: { "Content-Type": "text/event-stream" } })
}
```

#### 前端连接管理

```ts
// 前端 SSE 连接（EventSource 原生支持自动重连）
const es = new EventSource("/api/events/stream")

es.onmessage = (event) => {
  const data = JSON.parse(event.data)
  // 更新 UI：局中人状态、过堂发言、阶段流转...
}

es.onerror = () => {
  // 浏览器会自动重连（默认 3 秒间隔）
  // UI 显示"重新连接中..."
}
```

#### 断线恢复

SSE 断线后浏览器会自动重连，但重连期间可能错过事件。处理方式：

- 每个 SSE 事件带 `id` 字段（events 表的自增 ID）
- 重连时浏览器自动发送 `Last-Event-ID` header
- 服务端从该 ID 之后的事件开始推送，补上断线期间的数据
- 如果断线超过 5 分钟，前端放弃增量补推，改为全量刷新页面数据

#### 延迟预期

- 正常情况：Agent 产出 → 写入数据库 → SSE 推送 → 前端更新，**< 2 秒**
- 编排层繁忙时：可能延迟到 **5-10 秒**（等当前 tick 完成）
- 不承诺"实时"，承诺"准实时"——对展示网站来说足够了

### Agent 数据接口（两层架构）

**方案**：底层 Route Handler + 上层 OpenClaw Tool 封装

Agent 通过 OpenClaw tool 读写数据库，不需要知道 HTTP 细节。底层 API 同时服务于展示网站和管理后台。

```
Agent 调用 useTool("getMyStats")
    ↓
OpenClaw Tool 封装层（packages/openclaw/src/tools/）
    ↓ HTTP (localhost)
Route Handler（apps/site/src/app/api/pipeline/）
    ↓
PostgreSQL
```

安全策略：
- MVP 阶段：同机部署，API 监听 localhost，无需认证
- 生产阶段：写入接口用 HMAC session token，读取接口公开（数据本身要在展示网站展示）

详细设计见 [Agent工作区设计/API.md](./Agent工作区设计/API.md)。

### Agent 与 OpenClaw 集成

OpenClaw 中的 Agent 不是代码定义的函数，而是由 workspace 文件驱动的角色：

- **SOUL.md** 定义人格和行为准则
- **TOOLS.md** 定义可用工具和输出 schema
- **HEARTBEAT.md** 定义更鼓响时的任务
- **MEMORY.md** 存储成长档案

编排层通过修改 HEARTBEAT.md 来分配任务，Agent 通过 tool 调用来读写数据。

```
编排层检测到新物帖
    ↓
更新游商的 HEARTBEAT.md："有新物帖等待采风"
    ↓
更鼓响起，OpenClaw Gateway 唤醒游商
    ↓
游商读 HEARTBEAT.md → 调用 getProjectContext → 调用 web_search → 调用 submitScoutReport
    ↓
编排层收到产出 → 检查评分 → 决定进入过堂或封存
```

详细设计见 [Agent工作区设计/](./Agent工作区设计/)。

### 为什么选择这套组合

| 服务          | 理由                                            |
| ------------- | ----------------------------------------------- |
| VPS           | Next.js + OpenClaw Gateway 同机运行，API 走 localhost |
| Supabase/Neon | Serverless PostgreSQL，免运维，有 Realtime 能力 |
| Upstash Redis | Serverless Redis，按量付费，适合更鼓队列        |
| Cloudflare R2 | S3 兼容，无出口流量费，适合存截图和产物         |

> MVP 阶段 OpenClaw Gateway 和 Next.js 跑在同一台机器上，Agent 数据接口走 localhost，安全且简单。

## 安全考量

| 风险               | 措施                                                |
| ------------------ | --------------------------------------------------- |
| Agent 生成恶意代码 | 沙箱执行 + 试剑审查 + 回炉机制                      |
| 物帖注入           | 输入端 zod 校验 + Agent prompt 注入防护              |
| Agent 数据越权     | 写入接口 HMAC token + 幂等性 + 来源校验             |
| 部署到生产环境     | 鸣锣冒烟测试 + 自动回滚                             |
| 成本失控           | Agent 调用次数限制 + 预算告警 + 单造物令 72h 超时    |
| 数据泄露           | Agent 日志中脱敏 + 最小权限（读自己、写当前阶段）    |
