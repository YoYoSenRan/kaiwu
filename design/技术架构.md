## 关键技术决策

### 流水线引擎

**方案**：自建轻量状态机，不引入重型工作流引擎

理由：

- 阶段数固定（6个），不需要动态编排
- 需要深度定制每个阶段的行为
- OpenClaw 本身提供了 Agent 调度能力，引擎只需管状态流转

核心接口：

```ts
interface PipelineEngine {
  /** 启动新项目流水线 */
  start(keywordId: string): Promise<Project>

  /** 推进到下一阶段 */
  advance(projectId: string): Promise<Phase>

  /** 人类决策：批准/否决 */
  decide(projectId: string, decision: Decision): Promise<void>

  /** 获取项目当前状态 */
  getStatus(projectId: string): Promise<ProjectStatus>

  /** 重试失败的阶段 */
  retry(projectId: string, phaseId: string): Promise<void>
}
```

### 实时推送

**方案**：Server-Sent Events (SSE)

理由：

- 数据流向单一（服务端 → 客户端），不需要 WebSocket 的双向通信
- 实现简单，Next.js Route Handler 原生支持
- 自动重连
- Agent 状态变更、新事件通过 SSE 推送到展示网站

```ts
// app/api/events/stream/route.ts
export async function GET() {
  const stream = new ReadableStream({
    start(controller) {
      const listener = (event: PipelineEvent) => {
        controller.enqueue(`data: ${JSON.stringify(event)}\n\n`)
      }
      eventBus.subscribe(listener)
    },
  })

  return new Response(stream, { headers: { "Content-Type": "text/event-stream" } })
}
```

### Agent 与 OpenClaw 集成

```ts
// packages/openclaw/agents/scout.ts
import { defineAgent } from "@openclaw/sdk"

export const scoutAgent = defineAgent({
  name: "scout",
  description: "市场探测 Agent，负责关键词的多维度分析",
  tools: [webSearch, trendAnalysis, competitorScan],

  async execute(input: ScoutInput): Promise<ScoutReport> {
    // 1. 市场规模调研
    const market = await this.useTool("webSearch", { query: `${input.keyword} market size` })

    // 2. 竞品分析
    const competitors = await this.useTool("competitorScan", { keyword: input.keyword })

    // 3. 技术可行性评估
    const techFeasibility = await this.analyze("技术可行性", input)

    // 4. 生成综合报告
    return this.generateReport({ market, competitors, techFeasibility })
  },
})
```


### 为什么选择这套组合

| 服务          | 理由                                            |
| ------------- | ----------------------------------------------- |
| Vercel        | Next.js 最佳部署平台，SSR/SSG/ISR 全支持        |
| Supabase/Neon | Serverless PostgreSQL，免运维，有 Realtime 能力 |
| Upstash Redis | Serverless Redis，按量付费，适合 Agent 状态管理 |
| Cloudflare R2 | S3 兼容，无出口流量费，适合存截图和产物         |

## 安全考量

| 风险               | 措施                                    |
| ------------------ | --------------------------------------- |
| Agent 生成恶意代码 | 沙箱执行 + Inspector 审查 + 人工确认    |
| 关键词注入         | 输入端 zod 校验 + Agent prompt 注入防护 |
| 部署到生产环境     | 发布前人工确认 + 回滚机制               |
| 成本失控           | Agent 调用次数限制 + 预算告警           |
| 数据泄露           | Agent 日志中脱敏 + 最小权限原则         |
