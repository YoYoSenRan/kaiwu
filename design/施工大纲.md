# 施工大纲

## 总体原则

- **垂直切片**：每个阶段交付一个端到端可运行的能力，不逐层堆砌
- **先跑通再打磨**：先让一个物帖完整走完造物流，再优化展示体验
- **设计文档即参考**：每个施工步骤标注对应的设计文档位置，开发时直接查阅
- **每个阶段开工前再写详细施工文档**：本文件只是大纲，具体实现细节在开工时细化

## 施工节奏建议

| Phase | 预估周期 | 核心风险 |
|---|---|---|
| Phase 0 地基 | 1 周 | 无（基础设施搭建） |
| Phase 1 采风 | 1-2 周 | 游商 prompt 质量（采风报告是否有用） |
| Phase 2 过堂 | 2 周 | 辩论内容是否精彩（核心卖点） |
| Phase 3 造物志+记忆 | 1-2 周 | 展示体验是否吸引人 |
| Phase 4 绘图+锻造+试剑 | 2-3 周 | 匠人产出的代码质量（最大风险） |
| Phase 5 鸣锣 | 1 周 | 部署自动化 |
| Phase 6 打磨 | 持续 | — |

**总计约 8-11 周**完成 Phase 0-5，之后 Phase 6 持续迭代。

**建议**：Phase 0 完成后，用 2-3 个种子物帖跑通 Phase 1-2，验证采风和过堂的质量。如果质量不行，在这个阶段调 prompt 比后面再调成本低得多。
