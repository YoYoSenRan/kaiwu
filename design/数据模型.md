# 数据模型设计

## 核心实体关系

```
User ──1:N──→ Keyword ──1:1──→ Project ──1:N──→ Phase ──1:N──→ Task
  │                                │                │
  │                                │                └──1:N──→ Debate（过堂专用）
  │                                │
  └──M:N──→ Vote（on Keyword）     ├──1:1──→ Product（鸣锣后生成）
                                   └──1:N──→ Retrospective（复盘）

Agent ──1:N──→ AgentStat（属性快照）
Agent ──1:N──→ AgentMemory（经验记忆）
Agent ──1:N──→ AgentLog（全过程日志）

Event（事件流）── 关联 Project / Phase / Agent
```

---

## 表设计

### users — 用户

```sql
users
├── id              UUID PK
├── github_id       VARCHAR UNIQUE  -- GitHub 用户 ID
├── username        VARCHAR UNIQUE  -- 显示名（同步自 GitHub）
├── avatar_url      VARCHAR         -- 头像
├── github_stars    INTEGER         -- GitHub 星数（注册时快照，用于权重加成）
├── github_created  TIMESTAMP       -- GitHub 账号创建时间（防机器人）
├── last_submit_at  DATE            -- 最后一次提交物帖的日期（每天限 1 次）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### keywords — 物帖池

```sql
keywords
├── id              UUID PK
├── text            VARCHAR         -- 物帖文本（1~20 字）
├── reason          TEXT            -- 提交理由："为什么值得造"（20~200 字）
├── source          VARCHAR         -- 来源：user_submit / system_seed
├── submitted_by    UUID FK → users -- 提交者
├── score           INTEGER         -- 综合评分 0-100（采风后填入）
├── green_votes     INTEGER DEFAULT 0  -- 🟢 盖印（值得造）票数
├── red_votes       INTEGER DEFAULT 0  -- ⚪ 留白（不看好）票数
├── weight          FLOAT           -- 综合权重（由投票+时间衰减+提交者加成计算）
├── status          VARCHAR         -- pending / queued / scouting / in_pipeline / completed / dead
├── pre_scout_data  JSONB           -- 游商预采风缓存（自由活动时写入，正式采风时参考）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### votes — 投票

```sql
votes
├── id              UUID PK
├── user_id         UUID FK → users
├── keyword_id      UUID FK → keywords
├── stance          VARCHAR         -- green（盖印）/ red（留白）
├── created_at      TIMESTAMP
├── updated_at      TIMESTAMP
└── UNIQUE(user_id, keyword_id)     -- 每人每物帖限一票
```

### projects — 造物令

```sql
projects
├── id              UUID PK
├── keyword_id      UUID FK → keywords
├── name            VARCHAR         -- 器物名称（绘图阶段确定）
├── slug            VARCHAR UNIQUE  -- URL 友好标识
├── description     TEXT            -- 一句话描述
├── status          VARCHAR         -- 见状态枚举
├── current_phase   VARCHAR         -- scout / council / architect / builder / inspector / deployer
├── product_url     VARCHAR         -- 鸣锣后的访问地址
├── repo_url        VARCHAR         -- 代码仓库地址
├── started_at      TIMESTAMP       -- 进入造物流时间
├── launched_at     TIMESTAMP       -- 鸣锣时间
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

> 阶段产出（采风报告、裁决书、蓝图等）存在 `phases.output` 中，不在 projects 表冗余存储。

### phases — 阶段记录

```sql
phases
├── id              UUID PK
├── project_id      UUID FK → projects
├── type            VARCHAR         -- scout / council / architect / builder / inspector / deployer
├── status          VARCHAR         -- pending / in_progress / completed / failed / skipped
├── attempt         INTEGER DEFAULT 1  -- 第几次尝试（试剑回炉时递增）
├── input           JSONB           -- 该阶段的输入数据（上游产出的引用）
├── output          JSONB           -- 该阶段的输出数据（采风报告/裁决书/蓝图/审查报告/部署报告）
├── started_at      TIMESTAMP
├── completed_at    TIMESTAMP
├── duration_ms     INTEGER         -- 耗时（毫秒）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### debates — 过堂记录（论证阶段专用）

```sql
debates
├── id              UUID PK
├── phase_id        UUID FK → phases
├── round           INTEGER         -- 第几轮（1-4）
├── agent_id        VARCHAR FK → agents  -- shuike / zhengchen / zhangcheng
├── content         TEXT            -- 发言内容（Markdown）
├── citations       JSONB           -- 引用的数据来源
├── stance          VARCHAR         -- support / oppose / neutral
├── key_point       VARCHAR         -- 本轮核心论点（一句话）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### tasks — 任务（锻造阶段）

```sql
tasks
├── id              UUID PK
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases
├── title           VARCHAR
├── description     TEXT
├── status          VARCHAR         -- pending / in_progress / completed / blocked / cancelled
├── priority        INTEGER         -- 1=最高
├── assigned_agent  VARCHAR FK → agents  -- jiangren（匠人）
├── sub_role        VARCHAR         -- qi（器/前端）/ gu（骨/后端）/ yan（验/测试）
├── depends_on      UUID[]          -- 依赖的前置任务 ID
├── result          JSONB           -- 任务结果（commits、decisions 等）
├── started_at      TIMESTAMP
├── completed_at    TIMESTAMP
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### agents — 局中人注册表

```sql
agents
├── id              VARCHAR PK      -- youshang / shuike / zhengchen / zhangcheng / huashi / jiangren / shijian / mingluo
├── name            VARCHAR         -- 显示名称（游商 / 说客 / 诤臣 / ...）
├── title           VARCHAR         -- 司职（采风使 / 立论使 / 驳论使 / ...）
├── emoji           VARCHAR         -- 签名 emoji（🎒 / 🗣 / ⚔️ / ...）
├── stage_type      VARCHAR         -- 对应的造物流阶段
├── personality     JSONB           -- 性格描述、口头禅、说话风格
├── status          VARCHAR         -- idle / thinking / working / debating / blocked
├── current_project UUID FK → projects  -- 当前参与的造物令
├── level           INTEGER DEFAULT 1   -- 等级（1-5）
├── level_name      VARCHAR         -- 等级名（初出茅庐 / 略有小成 / ...）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### agent_stats — 局中人属性（每次造物令完成后更新）

```sql
agent_stats
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── stat_key        VARCHAR         -- 属性名：嗅觉 / 口才 / 洞察 / 公正 / 精微 / 手艺 / 眼力 / 稳妥 / ...
├── raw_value       FLOAT           -- 原始数据值（如准确率 0.87）
├── star_level      INTEGER         -- 星级 1-5
├── sample_size     INTEGER         -- 基于多少次造物令计算
├── updated_at      TIMESTAMP
└── UNIQUE(agent_id, stat_key)
```

### agent_memories — 局中人经验记忆

```sql
agent_memories
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── project_id      UUID FK → projects    -- 来源造物令
├── type            VARCHAR         -- pattern（模式）/ lesson（教训）/ mistake（失误）
├── content         TEXT            -- 经验内容
├── domain          VARCHAR         -- 领域标签（工具类 / 社交类 / 电商类 / ...）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### agent_logs — 局中人日志（全过程追溯的核心）

```sql
agent_logs
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases
├── task_id         UUID FK → tasks     -- 可选
├── type            VARCHAR         -- thought / action / decision / error
├── content         TEXT            -- 日志内容
├── metadata        JSONB           -- 附加数据（搜索结果、代码 diff 等）
├── visibility      VARCHAR         -- public / internal（是否在展示网站显示）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### products — 器物（鸣锣后生成）

```sql
products
├── id              UUID PK
├── project_id      UUID FK → projects
├── name            VARCHAR
├── tagline         VARCHAR         -- 一句话标语
├── description     TEXT            -- 详细描述
├── url             VARCHAR         -- 访问地址
├── repo_url        VARCHAR         -- 源码地址
├── screenshot_urls VARCHAR[]       -- 截图
├── tech_stack      VARCHAR[]       -- 技术栈标签
├── metrics         JSONB           -- 运行指标：uptime、访问量等
├── story_published BOOLEAN         -- 造物志是否已发布
├── launched_at     TIMESTAMP
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### retrospectives — 复盘记录

```sql
retrospectives
├── id              UUID PK
├── project_id      UUID FK → projects
├── trigger_day     INTEGER         -- 第几天的复盘（7 / 30 / 90）
├── metrics         JSONB           -- 器物的实际表现数据（访问量、留存率等）
├── verdicts        JSONB           -- 各局中人当初的判断 vs 实际结果
├── stat_changes    JSONB           -- 本次复盘导致的属性变更
├── summary         TEXT            -- 复盘志正文（发布到展示网站）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### events — 事件流（时间线回放的数据源）

```sql
events
├── id              UUID PK
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases     -- 可选
├── agent_id        VARCHAR FK → agents  -- 可选
├── type            VARCHAR         -- 见事件类型枚举
├── title           VARCHAR         -- 事件标题（用于时间线展示）
├── detail          JSONB           -- 事件详情
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

## 状态枚举

### 造物令状态（Project Status）

```
scouting        -- 采风中
debating        -- 过堂中
planning        -- 绘图中
building        -- 锻造中
inspecting      -- 试剑中
deploying       -- 鸣锣中
launched        -- 已开物（器物问世）
dead            -- 封存（否决 / 失败 / 超时）
```

### 事件类型（Event Type）

```
-- 物帖
keyword_submitted       -- 物帖投入
vote_cast              -- 盖印/留白

-- 采风
scout_started          -- 游商出发
scout_report_ready     -- 采风报告就绪
scout_rejected         -- 采风评分不足，封存

-- 过堂
debate_round_start     -- 过堂轮次开始
debate_speech          -- 过堂发言（说客/诤臣）
debate_verdict         -- 掌秤裁决
epitaph_written        -- 封存辞撰写

-- 绘图
blueprint_created      -- 蓝图完成

-- 锻造
task_assigned          -- 任务分配
task_completed         -- 任务完成
code_committed         -- 代码提交

-- 试剑
review_issue_found     -- 试剑发现问题
review_passed          -- 试剑通过
review_failed          -- 试剑不通过，回炉

-- 鸣锣
deploy_started         -- 开始部署
deploy_completed       -- 部署完成
smoke_test_failed      -- 冒烟测试失败，回滚
product_launched       -- 锣响三声，器物问世

-- 属性与复盘
stat_updated           -- 局中人属性变化
retrospective_created  -- 复盘志发布
```

## 索引建议

```sql
-- 用户与投票
CREATE UNIQUE INDEX idx_users_github_id ON users(github_id);
CREATE UNIQUE INDEX idx_votes_user_keyword ON votes(user_id, keyword_id);

-- 物帖排序
CREATE INDEX idx_keywords_status_weight ON keywords(status, weight DESC);
CREATE INDEX idx_keywords_submitted_by ON keywords(submitted_by);

-- 造物令与阶段
CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_current_phase ON projects(current_phase);
CREATE INDEX idx_phases_project_type ON phases(project_id, type);

-- 过堂记录
CREATE INDEX idx_debates_phase_round ON debates(phase_id, round);

-- 任务
CREATE INDEX idx_tasks_project_status ON tasks(project_id, status);
CREATE INDEX idx_tasks_assigned_agent ON tasks(assigned_agent, status);

-- 局中人属性与记忆
CREATE UNIQUE INDEX idx_agent_stats_key ON agent_stats(agent_id, stat_key);
CREATE INDEX idx_agent_memories_agent_domain ON agent_memories(agent_id, domain);

-- 事件与日志
CREATE INDEX idx_events_project_id ON events(project_id);
CREATE INDEX idx_events_type ON events(type);
CREATE INDEX idx_events_created_at ON events(created_at);
CREATE INDEX idx_agent_logs_project_id ON agent_logs(project_id);
CREATE INDEX idx_agent_logs_agent_id ON agent_logs(agent_id);

-- 复盘
CREATE INDEX idx_retrospectives_project ON retrospectives(project_id);
```
