# 数据模型设计

## 核心实体关系

```
User ──1:N──→ Keyword ──1:1──→ Project ──1:N──→ Phase ──1:N──→ Task
  │                                │                │
  │                                │                └──1:N──→ AgentLog
  │                                │
  └──M:N──→ Vote（on Keyword）     └──1:1──→ Product（上线后生成）

Agent ──M:N──→ Task（通过 AgentTask）
Agent ──1:N──→ AgentLog
```

---

## 表设计

### users — 用户

```sql
users
├── id              UUID PK
├── github_id       VARCHAR UNIQUE  -- GitHub 用户 ID
├── username        VARCHAR UNIQUE  -- 显示名（同步自 GitHub）
├── avatar_url      VARCHAR         -- 头像
├── github_stars    INTEGER         -- GitHub 星数（注册时快照，用于权重加成）
├── github_created  TIMESTAMP       -- GitHub 账号创建时间（防机器人）
├── last_submit_at  DATE            -- 最后一次提交关键词的日期（每天限 1 次）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### keywords — 关键词池

```sql
keywords
├── id              UUID PK
├── text            VARCHAR         -- 关键词文本（1~20 字）
├── reason          TEXT            -- 提交理由："为什么值得做"（20~200 字）
├── source          VARCHAR         -- 来源：user_submit / system_seed
├── submitted_by    UUID FK → users -- 提交者
├── score           INTEGER         -- 综合评分 0-100（探测后填入）
├── green_votes     INTEGER DEFAULT 0  -- 🟢 值得做 票数
├── red_votes       INTEGER DEFAULT 0  -- 🔴 不看好 票数
├── weight          FLOAT           -- 综合权重（由投票+时间衰减+提交者加成计算）
├── status          VARCHAR         -- pending / queued / scouting / in_pipeline / completed / dead
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### votes — 投票

```sql
votes
├── id              UUID PK
├── user_id         UUID FK → users
├── keyword_id      UUID FK → keywords
├── stance          VARCHAR         -- green / red
├── created_at      TIMESTAMP
├── updated_at      TIMESTAMP
└── UNIQUE(user_id, keyword_id)     -- 每人每关键词限一票
```

### projects — 项目

```sql
projects
├── id              UUID PK
├── keyword_id      UUID FK → keywords
├── name            VARCHAR         -- 项目名称（规划阶段确定）
├── slug            VARCHAR UNIQUE  -- URL 友好标识
├── description     TEXT            -- 一句话描述
├── status          VARCHAR         -- 见状态机
├── current_phase   VARCHAR         -- scout / council / architect / builder / inspector / deployer
├── scout_report    JSONB           -- 探测报告
├── council_verdict JSONB           -- 论证决议
├── architecture    JSONB           -- 技术架构方案
├── product_url     VARCHAR         -- 上线后的访问地址
├── repo_url        VARCHAR         -- 代码仓库地址
├── verdict         VARCHAR         -- approved / rejected / suspended
├── verdict_reason  TEXT            -- 决策理由
├── started_at      TIMESTAMP       -- 进入流水线时间
├── launched_at     TIMESTAMP       -- 上线时间
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### phases — 阶段记录

```sql
phases
├── id              UUID PK
├── project_id      UUID FK → projects
├── type            VARCHAR         -- scout / council / architect / builder / inspector / deployer
├── status          VARCHAR         -- pending / in_progress / completed / failed / skipped
├── input           JSONB           -- 该阶段的输入数据
├── output          JSONB           -- 该阶段的输出数据
├── started_at      TIMESTAMP
├── completed_at    TIMESTAMP
├── duration_ms     INTEGER         -- 耗时（毫秒）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### debates — 辩论记录（论证阶段专用）

```sql
debates
├── id              UUID PK
├── phase_id        UUID FK → phases
├── round           INTEGER         -- 第几轮
├── agent_role      VARCHAR         -- optimist / skeptic / pragmatist
├── content         TEXT            -- 发言内容（Markdown）
├── citations       JSONB           -- 引用的数据来源
├── stance          VARCHAR         -- support / oppose / neutral
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### tasks — 任务（构建阶段）

```sql
tasks
├── id              UUID PK
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases
├── title           VARCHAR
├── description     TEXT
├── status          VARCHAR         -- pending / in_progress / completed / blocked / cancelled
├── priority        INTEGER         -- 1=最高
├── assigned_agent  VARCHAR         -- 分配给哪个 Agent
├── depends_on      UUID[]          -- 依赖的前置任务 ID
├── result          JSONB           -- 任务结果
├── started_at      TIMESTAMP
├── completed_at    TIMESTAMP
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### agents — Agent 注册表

```sql
agents
├── id              VARCHAR PK      -- scout / optimist / skeptic / ...
├── name            VARCHAR         -- 显示名称
├── role            VARCHAR         -- 角色类型
├── avatar_url      VARCHAR         -- 头像地址
├── personality     JSONB           -- 性格描述、口头禅等
├── status          VARCHAR         -- idle / thinking / working / discussing / blocked
├── current_task_id UUID FK → tasks -- 当前执行的任务
├── stats           JSONB           -- 统计数据：完成任务数、参与项目数等
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### agent_logs — Agent 日志（全过程追溯的核心）

```sql
agent_logs
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases
├── task_id         UUID FK → tasks     -- 可选
├── type            VARCHAR         -- thought / action / message / decision / error
├── content         TEXT            -- 日志内容
├── metadata        JSONB           -- 附加数据（代码 diff、搜索结果等）
├── visibility      VARCHAR         -- public / internal（是否在展示网站显示）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### products — 已上线产品

```sql
products
├── id              UUID PK
├── project_id      UUID FK → projects
├── name            VARCHAR
├── tagline         VARCHAR         -- 一句话标语
├── description     TEXT            -- 详细描述
├── url             VARCHAR         -- 访问地址
├── repo_url        VARCHAR         -- 源码地址
├── screenshot_urls VARCHAR[]       -- 截图
├── tech_stack      VARCHAR[]       -- 技术栈标签
├── metrics         JSONB           -- 运行指标：uptime、访问量等
├── story_published BOOLEAN         -- 产品故事是否已发布
├── launched_at     TIMESTAMP
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

### events — 事件流（时间线回放的数据源）

```sql
events
├── id              UUID PK
├── project_id      UUID FK → projects
├── phase_id        UUID FK → phases
├── agent_id        VARCHAR FK → agents    -- 可选
├── type            VARCHAR         -- 见事件类型枚举
├── title           VARCHAR         -- 事件标题（用于时间线展示）
├── detail          JSONB           -- 事件详情
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```

## 状态枚举

### 项目状态（Project Status）

```
scouting        -- 探测中
debating        -- 论证中
planning        -- 规划中
building        -- 构建中
inspecting      -- 审查中
deploying       -- 发布中
launched        -- 已上线
dead            -- 已否决 / 失败（归入墓地）
```

### 事件类型（Event Type）

```
keyword_submitted       -- 关键词提交
scout_started          -- 开始探测
scout_report_ready     -- 探测报告就绪
debate_round_start     -- 辩论轮次开始
debate_speech          -- 辩论发言
debate_verdict         -- 辩论裁决
vote_cast              -- 用户投票
plan_created           -- 规划生成
task_assigned          -- 任务分配
task_completed         -- 任务完成
code_committed         -- 代码提交
review_issue_found     -- 审查发现问题
review_passed          -- 审查通过
deploy_started         -- 开始部署
deploy_completed       -- 部署完成
product_launched       -- 产品上线
project_rejected       -- 项目否决
```

## 索引建议

```sql
-- 用户与投票
CREATE UNIQUE INDEX idx_users_github_id ON users(github_id);
CREATE UNIQUE INDEX idx_votes_user_keyword ON votes(user_id, keyword_id);

-- 关键词排序
CREATE INDEX idx_keywords_status_weight ON keywords(status, weight DESC);
CREATE INDEX idx_keywords_submitted_by ON keywords(submitted_by);

-- 项目与阶段
CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_current_phase ON projects(current_phase);

-- 事件与日志
CREATE INDEX idx_events_project_id ON events(project_id);
CREATE INDEX idx_events_created_at ON events(created_at);
CREATE INDEX idx_agent_logs_project_id ON agent_logs(project_id);
CREATE INDEX idx_agent_logs_agent_id ON agent_logs(agent_id);
CREATE INDEX idx_debates_phase_id ON debates(phase_id, round);
CREATE INDEX idx_tasks_project_status ON tasks(project_id, status);
```
