### 页面视觉基调

```
与其他页面的差异:
  - 更多 --font-mono 使用（标题辅助文字、数据标注）
  - 背景允许出现微弱网格线（grid pattern, opacity 3%）
  - Section 分隔用虚线而非实线: border-top 1px dashed --border
  - 代码块和技术术语频繁出现，不做简化
  - 整体"技术博客"的阅读节奏，比其他页面更长、更有深度
```

### 6 个阶段的叙事内容要点

```
采风: 游商出门跑市面，只带回数据。评分决定生死。
过堂: 说客和诤臣当庭辩论，掌秤最后落锤。最有戏剧张力的阶段。
绘图: 画师把裁决书变成蓝图——功能、架构、任务拆解。
锻造: 匠人抡起锤子，一锤一锤把代码敲出来。最耗时的阶段。
试剑: 试剑官挑毛病——代码质量、安全、性能，最多给三次返工机会。
鸣锣: 部署上线，冒烟测试通过，敲锣宣告"器物问世"。
```

### 交互说明

- 默认选中第一个阶段（采风），展示其详情卡
- 点击其他阶段节点：详情卡切换，旧卡 fadeOut → 新卡 fadeIn
- 流程节点之间的连接线有微弱的流动动画（dash-offset，像传送带在动）

### 响应式适配

```
桌面 (≥ 1024px): 流程图水平排列，6 节点一行
平板 (768-1023px): 流程图水平排列，节点缩小到 64x64
移动 (< 768px):
  流程图改为垂直排列:
  节点竖向排列，连接线变竖线
  详情卡全宽，紧跟在选中节点下方
```
