# 角色属性系统

## 设计原则

- **属性来自实战**：不是预设的静态数值，而是从每次采风、过堂、锻造中统计得来
- **每个角色的属性和他的职责强相关**：不套用"力量/敏捷/智力"通用模板
- **属性名用开物局的语言**：不叫"准确率"，叫"嗅觉"
- **有成长感**：参与的造物令越多，属性越高——但不是单调递增，复盘可能导致某些属性下降
- **可展示**：属性值是角色介绍页上最吸引人的信息之一

## 各角色属性

### 🎒 游商

| 属性     | 含义                 | 计算方式                                |
| -------- | -------------------- | --------------------------------------- |
| **嗅觉** | 市场判断准不准       | 采风评分与器物最终表现的相关度          |
| **脚力** | 采风速度             | 平均采风耗时                            |
| **见闻** | 阅历深度             | 累计经手物帖数                          |
| **慧眼** | 发现别人看不到的机会 | 采风报告中被说客/画师引用的独特洞察次数 |

**特殊属性**：

- **领域专长**：工具类 ★★★★☆ / 社交类 ★★☆☆☆ / 电商类 ★★★☆☆
  - 根据各领域采风准确率自动计算

### 🗣 说客

| 属性     | 含义             | 计算方式                      |
| -------- | ---------------- | ----------------------------- |
| **口才** | 说服力           | 过堂中被掌秤采纳的论点比例    |
| **博引** | 引经据典的能力   | 平均每轮引用的有效案例/数据数 |
| **韧性** | 被反驳后还能翻盘 | 第 3 轮及之后反击成功次数     |
| **信誉** | 说的话靠不靠谱   | 他支持的造物令最终成功的比例  |

**特殊属性**：

- **最佳搭档**：和哪位诤臣搭配时通过率最高
- **克星**：最常被哪位诤臣的哪类论点驳倒

### ⚔️ 诤臣

| 属性     | 含义               | 计算方式                                               |
| -------- | ------------------ | ------------------------------------------------------ |
| **洞察** | 找到致命漏洞的能力 | 提出的质疑中被掌秤采纳为关键因素的比例                 |
| **一击** | 一句话定生死       | 单轮发言直接导致封存的次数                             |
| **公心** | 不是为反对而反对   | 被说客说服后改变立场的次数（越高越好——说明他真的在听） |
| **先见** | 预判风险的准确度   | 他提出的风险在后续阶段/复盘中被验证的比例              |

**特殊属性**：

- **封存率**：经他手的过堂中物帖被封存的比例
- **宿敌战绩**：vs 说客的交锋胜负记录

### ⚖️ 掌秤

| 属性     | 含义             | 计算方式                               |
| -------- | ---------------- | -------------------------------------- |
| **公正** | 裁决的准确度     | 裁决结果与器物最终命运的吻合度         |
| **果断** | 不拖泥带水       | 平均裁决用时（从最后一轮辩论到出裁决） |
| **权衡** | 综合分析的深度   | 裁决中引用双方论点的平衡度             |
| **远见** | 附带条件是否合理 | "有条件造"的条件最终被验证为必要的比例 |

**特殊属性**：

- **翻案率**：他放行的造物令最终失败的比例（越低越好）
- **封存准确率**：他封存的物帖，同类产品后来也没人做成的比例

### 🖌 画师

| 属性     | 含义             | 计算方式                                    |
| -------- | ---------------- | ------------------------------------------- |
| **精微** | 蓝图的细致程度   | 因蓝图问题导致匠人返工的次数（越少越好）    |
| **全局** | 系统设计能力     | 蓝图中预见到的技术风险 / 实际遇到的技术风险 |
| **化繁** | 把复杂问题简单化 | 蓝图的任务拆解数与最终实际任务数的偏差      |
| **效率** | 绘图速度         | 平均绘图耗时                                |

### 🔨 匠人

| 属性       | 含义                 | 计算方式                               |
| ---------- | -------------------- | -------------------------------------- |
| **手艺**   | 代码/产出质量        | 试剑阶段发现的🔴必修问题数（越少越好） |
| **耐力**   | 持续产出能力         | 单次造物令中最长连续工作的更鼓数       |
| **巧思**   | 超出蓝图预期的好设计 | 试剑人标注为"超出预期"的次数           |
| **返工率** | 被试剑打回的频率     | 被打回重做的任务比例（越低越好）       |

**特殊属性**：

- **擅长领域**：前端 / 后端 / 全栈，根据完成任务类型统计

### 🗡 试剑

| 属性     | 含义                   | 计算方式                                     |
| -------- | ---------------------- | -------------------------------------------- |
| **眼力** | 发现问题的能力         | 每次试剑发现的有效问题数                     |
| **精准** | 不误判                 | 标记的问题中确实是问题的比例（vs 误报）      |
| **严苛** | 标准是否够高           | 经他放行的器物在坊间出问题的概率（越低越好） |
| **公道** | 给出的修复建议是否可行 | 匠人按建议修复的一次通过率                   |

### 🔔 鸣锣

| 属性     | 含义                 | 计算方式                           |
| -------- | -------------------- | ---------------------------------- |
| **稳妥** | 上线后不出事         | 鸣锣后 24 小时内回滚率（越低越好） |
| **利落** | 部署速度             | 平均鸣锣耗时                       |
| **周全** | checklist 执行完整度 | 部署步骤的完成率                   |
| **兜底** | 出了事能兜住         | 回滚成功率                         |

## 属性等级

每个属性用五级评定，对应展示时的视觉：

| 等级  | 展示 | 含义     |
| ----- | ---- | -------- |
| ★☆☆☆☆ | 一星 | 初出茅庐 |
| ★★☆☆☆ | 二星 | 略有小成 |
| ★★★☆☆ | 三星 | 独当一面 |
| ★★★★☆ | 四星 | 炉火纯青 |
| ★★★★★ | 五星 | 登峰造极 |

**升星条件**：综合该属性的原始数据 + 经手的造物令数量。不是简单的"做得多就升"——准确率下降时也会降星。

## 经验循环

```
造物令完成（鸣锣 or 封存）
  │
  ▼
结果反馈给所有参与的局中人
  │
  ├── 游商：我的采风评分准不准？
  │         → 更新「嗅觉」属性
  │
  ├── 说客：我支持的物帖成功了吗？
  │         → 更新「信誉」属性
  │
  ├── 诤臣：我提的风险出现了吗？
  │         → 更新「先见」属性
  │
  ├── 掌秤：我的裁决对了吗？
  │         → 更新「公正」属性
  │
  └── ...其他角色类推
```

### 复盘机制

器物鸣锣上线后，每隔一段时间（7天、30天、90天）自动触发复盘：

```
复盘内容：
├── 器物在坊间表现如何？（访问量、留存率、用户反馈）
├── 当初说客说的机会验证了吗？
├── 诤臣提的风险出现了吗？
├── 掌秤的条件合理吗？
├── 画师的蓝图有没有遗漏？
└── 匠人的手艺经住考验了吗？

复盘输出：
├── 属性值调整
├── 经验沉淀（成功/失败的模式记录到角色的记忆中）
└── 一篇「复盘志」发布到展示网站（"三个月后回头看，谁说对了？"）
```

**复盘志是展示网站的优质内容**——"当初说客信誓旦旦说市场很大，三个月后来看，他说对了吗？"

## 属性的实际作用（不只是展示）

属性值不只是好看，它会真正影响 Agent 的行为：

| 属性           | 实际影响                                         |
| -------------- | ------------------------------------------------ |
| 游商的领域专长 | 采风时，擅长领域的物帖会自动引用更多历史数据     |
| 说客的信誉     | 信誉高时，他的论点在过堂中权重更大               |
| 诤臣的先见     | 先见高时，他提出的风险会被掌秤更认真对待         |
| 掌秤的公正     | 公正度高时，他的裁决不需要额外条件               |
| 画师的精微     | 精微度高时，蓝图可以更简洁（匠人需要的细节更少） |
| 匠人的手艺     | 手艺高时，试剑阶段可适当降低审查严格度           |
| 试剑的精准     | 精准度高时，他标记的🔴问题不需要二次确认         |

## 数据模型补充

```sql
-- 角色属性快照（每次造物令完成后更新）
agent_stats
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── stat_key        VARCHAR         -- 属性名：嗅觉 / 口才 / 洞察 / ...
├── raw_value       FLOAT           -- 原始数据值（如准确率 0.87）
├── star_level      INTEGER         -- 星级 1-5
├── sample_size     INTEGER         -- 基于多少次造物令计算
├── updated_at      TIMESTAMP
└── UNIQUE(agent_id, stat_key)

-- 复盘记录
retrospectives
├── id              UUID PK
├── project_id      UUID FK → projects
├── trigger_day     INTEGER         -- 第几天的复盘（7/30/90）
├── metrics         JSONB           -- 器物的实际表现数据
├── verdicts        JSONB           -- 各角色当初的判断 vs 实际结果
├── stat_changes    JSONB           -- 本次复盘导致的属性变更
├── summary         TEXT            -- 复盘志正文
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP

-- 角色经验记忆
agent_memories
├── id              UUID PK
├── agent_id        VARCHAR FK → agents
├── project_id      UUID FK → projects    -- 来源造物令
├── type            VARCHAR         -- pattern / lesson / mistake
├── content         TEXT            -- 经验内容
├── domain          VARCHAR         -- 领域标签（电商/工具/社交/...）
├── created_at      TIMESTAMP
└── updated_at      TIMESTAMP
```
