# 展示网站页面设计

## 设计理念

- **网站是活的**：不是静态展示页，而是一个有实时心跳的生命体——此刻正在发生什么、谁在争吵、谁在写代码
- **先有情绪，后有信息**：访客第一眼应该感到好奇和兴奋，而不是在读一份技术文档
- **每个页面讲一个故事**：首页 = "欢迎来到开物局"，故事页 = "一张物帖的一生"，Agent 页 = "认识这群局中人"
- **失败和成功同样精彩**：被否决的项目不是灰暗的角落，而是有独立叙事价值的内容

## 页面结构

```
/                        首页（Hero + 概览）
/stories                 造物志列表
/stories/:id             单个器物的完整造物志

/pipeline                实时造物坊
/architecture            技术架构介绍
/about                   关于项目
```

## 首页（/）

首页只有两个核心模块：**开物局全景**和**群聊记录**。访客一进来就看到一个活的开物局，往下滚就是局中人的实时对话。

### 模块一：开物局全景 — 横向卷轴可视化

**设计思路**：首页第一屏是一幅**横向长卷**——像《清明上河图》一样，从左到右展开整个开物局。访客看到的不是文字介绍，而是一个有人在走动、有炉火在烧、有人在吵架的**活的场景**。

**视觉风格**：水墨线条的建筑 + 简笔人物，墨色背景上用淡墨勾勒建筑轮廓。人物是简笔水墨小人，寥寥几笔但能看出动作。不是 3D 渲染，不是像素风，是**有留白的水墨长卷**。

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  ◀ 拖动浏览                    开 物 局                           下次更鼓 18:30 ▶  │
│                                                                                     │
│  ┌─ 物帖墙 ──┐  ┌── 过堂 ──────────┐  ┌─ 画室 ─┐  ┌─ 锻造坊 ─┐  ┌ 试剑台 ┐  ┌ 鸣锣 ┐
│  │            │  │                  │  │        │  │          │  │        │  │      │
│  │  ┌──┐     │  │  🗣      ⚔️      │  │        │  │  🔨      │  │        │  │      │
│  │  │帖│ 🎒  │  │  说客    诤臣    │  │  🖌    │  │  炉火    │  │  🗡    │  │  🔔  │
│  │  │帖│ →   │  │    ⚔️ 激辩中     │  │  💤    │  │  ⚡锻造中 │  │  💤    │  │  💤  │
```

#### 场景细节

每个区域有自己的**微动画**和**实时状态**：

| 区域 | 场景 | 动画 | 实时数据 |
|---|---|---|---|
| 物帖墙 | 墙上贴着排队的物帖，游商站在旁边 | 新物帖飘入时有纸张飘动效果 | 排队数量、游商状态 |
| 过堂 | 说客和诤臣面对面，掌秤居中 | 发言时对应人物有说话动作，局势条实时变化 | 当前轮次、局势比例 |
| 画室 | 画师在桌前，墙上挂着蓝图 | 绘图时有笔触动画 | 蓝图进度 |
| 锻造坊 | 匠人在炉前，炉火通明 | 炉火跳动 + 锤击火星粒子 | 当前任务、完成进度 |
| 试剑台 | 试剑人持剑站立 | 试剑时有挥剑动作 | 问题数量 |
| 鸣锣台 | 铜锣 + 鸣锣人 | 鸣锣时圆形波纹扩散 | 部署状态 |
| 器物坊 | 陈列架上摆着已完成的器物 | 新器物上架时有光泽闪烁 | 器物数量 |
| 封存阁 | 安静的阁楼，卷轴整齐排列 | 新封存时卷轴缓慢收拢飘入 | 封存数量 |

#### 人物状态气泡

每个局中人头顶有一个**状态气泡**，显示当前活动描述（来自 agents.activity）：

```
     ┌─────────────────────────────┐
     │ "22% 的留存率说明市场上      │
     │  没有合格的产品"             │
     └──────────────┬──────────────┘
                    │
                   🗣 说客 ⚔️ 激辩中
```

- 气泡内容是角色的最新发言或活动描述
- 气泡样式用角色专属色（说客暖色、诤臣冷色、掌秤金色...）
- 空闲的角色气泡显示闲聊状态（"在擦锤子"、"泡了杯茶"）
- 点击人物跳转到该角色的详情页

#### 交互

- **左右拖动/滑动**浏览整个长卷（移动端手势，桌面端拖拽或滚轮）
- **点击人物**：跳转到角色详情页
- **点击物帖墙**：跳转到物帖墙页
- **点击器物坊/封存阁**：跳转到造物志列表
- **hover 人物**：气泡放大，显示完整的活动描述
- 页面右上角显示**更鼓倒计时**："下次更鼓 18:30"

#### 技术实现方向

- 建筑和场景用 **SVG** 绘制（水墨线条风格，可缩放）
- 人物用 **CSS 动画** 或 **Lottie** 做微动画
- 不用 3D 引擎（Three.js 太重，且不符合水墨风格）
- 状态数据通过 **SSE** 实时推送
- 移动端横向滚动，桌面端可以一屏展示（如果屏幕够宽）

#### 无数据时的空状态

开物局刚启动、还没有任何造物令时：

```
长卷中所有房间都亮着灯，但没有人在干活。
物帖墙空空的，上面写着："还没有物帖。说一个关键词，看它的命运。"
八个局中人分散在各自的房间里，做着闲事。
游商在门口张望，说客在翻旧书，诤臣在磨刀...
```

### 模块二：群聊记录 — 局中人的实时对话

**设计思路**：长卷下方是一个**群聊记录列表**——所有局中人的发言按时间顺序排列，像微信群聊一样。访客往下滚就能看到局中人在说什么。

**这不是某个造物令的对话流**（那个在 `/stories/:id/flow`），而是**开物局的公共频道**——所有造物令的公开发言都汇聚在这里，按时间排序。

```

│                                                             │
│  开物局 · 此刻                                    实时 🔴   │
│                                                             │

│                                                             │
│  18:30  更鼓第 48 声                                        │
│                                                             │
│  18:31                                                      │
│  ┌─ 🗣 说客 ──────────────────────────────────────┐        │
│  │ "好问题。但你忽略了一个数据：现有记账 App 的      │        │
│  │  30 天留存率只有 22%。"                           │        │
│  │                                    「极简记账」   │        │
│  └────────────────────────────────────────────────┘        │
│                                                             │
│  18:33                                                      │
│                ┌─ ⚔️ 诤臣 ─────────────────────────┐       │
│                │ "22% 的留存率恰恰说明用户不需要     │       │
│                │  这类产品。"                        │       │
│                │                      「极简记账」   │       │

│                                                             │

│  ┌─ 🎒 游商 ──────────────────────────────────────┐        │
│  │ 顺路看了一眼「宠物心理咨询」，竞品不多，          │        │
│  │ 有意思。                                         │        │

│  │ "T-005 搞定了。又是三更天，又是居中的问题。"      │        │
│  │                                    「极简记账」   │        │
```

#### 群聊记录的特点

- **跨造物令**：所有造物令的公开发言都在这里，每条消息右下角标注来源（「极简记账」/ 自由活动）
- **按真实时间排序**：不按阶段分组，按发言时间流淌
- **实时更新**：SSE 推送新消息，像聊天窗口一样追加到顶部
- **更鼓标记**：每次更鼓响起在时间线中插入一条分隔线（"更鼓第 48 声"）
- **日期分隔**：按天分组（今天 / 昨天 / 3月15日...）
- **无限滚动**：往下滚加载更早的记录

#### 消息样式

- 说客的气泡靠左，暖色底
- 诤臣的气泡靠右，冷色底
- 掌秤的裁决有特殊样式（印章 + 金边）
- 游商的自由活动消息有"自由活动 🎒"标签，和造物流消息区分
- 系统消息（阶段转换、更鼓）居中小字

#### 筛选

顶部有简单的筛选：

- **全部**（默认）
- **只看过堂**（最精彩的辩论）
- **只看某个造物令**（跟踪一个故事）
- **只看某个角色**（关注某个局中人）

#### 数据来源

群聊记录从以下数据组装，按 `created_at` 统一排序：

| 来源 | 条件 | 显示内容 |
|---|---|---|
| agent_logs | visibility = 'public' | 局中人的思考和感想 |
| debates | 全部 | 过堂发言原文 |
| events | type 含 verdict / launched / rejected | 裁决、鸣锣、封存等关键节点 |
| agent_logs | type = 'free_activity' 或 'insight' | 游商的自由活动 |

**不展示**：visibility = 'internal' 的日志、技术细节、JSON 输出。

## 造物志（/stories）

**页面定位**：这是网站的核心内容页——每个创意从诞生到结局的完整纪录片。

**设计细节**：

- 已开物的卡片有明亮的主色调，带器物截图
- 进行中的卡片有微弱的脉冲动画，暗示"它还活着"
- 被否决的卡片用低饱和度色调，但"致命一击"引文用亮色高亮——失败也要有高光
- 每张卡片有一句**钩子文案**，让人想点进去看完整故事

### 详情页（/stories/:id）— 沉浸式造物志

**核心体验**：Apple 产品页风格的滚动叙事，不是传统的时间线列表。

每个阶段是一个"章节"，随着滚动逐步展开：

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                     极简记账                                 │
│              一个关键词的 72 小时旅程                          │
│                                                             │
│         由 @someone 提交于 2026-03-08 14:00                  │
│                                                             │
│  ┌─ 快速概览 ────────────────────────────────────────┐      │
│  │  提交 → 采风(28min) → 过堂(1h42min) →             │      │
│  │  绘图(55min) → 锻造(16h) → 试剑(2h) → 鸣锣(34min) │      │
```

### 被封存物帖的终章 — 封存辞

**设计思路**：被否决不是"项目页面变灰"，而是有一个专属的、有尊严的结局。

**关键设计**：

- "享年"这个概念：用项目在流水线上存活的时间来表达，有黑色幽默感
- 封存辞由掌秤撰写：一句话总结为什么它不值得做
- "致命一击"：标注是哪个 Agent 在哪一轮说了决定性的话
- "这个创意有救吗？"：允许用户提交改进版关键词，形成"复活"机制

## 造物令对话流（/stories/:id/flow）

**页面定位**：一个造物令中所有局中人交流的完整记录。像群聊的历史记录一样，**按真实发言时间排序**，从第一条到最后一条。从造物志详情页点击"查看完整对话流"进入。

**不是按阶段分块，而是按时间流淌。** 阶段转换只是时间线中的一条分隔线，不是分组依据。访客能感受到真实的节奏——游商 28 分钟就回来了，说客和诤臣吵了 40 分钟，匠人闷头干了一整夜。

**核心原则**：**只展示角色视角，不暴露技术细节。** 访客看到的是局中人在说话、在思考、在做决定——不是 API 调用、JSON 输出、阶段流转代码。

### 什么该展示，什么不该展示

| ✅ 展示 | ❌ 不展示 |
|---|---|
| 游商的采风观察和评价 | web_search 的 API 调用参数 |
| 说客和诤臣的辩论原文 | submitDebateSpeech 的 JSON schema |
| 掌秤的裁决理由 | 阶段流转的状态机逻辑 |
| 画师的设计思路和任务拆解 | 技术栈选型的代码细节 |
| 匠人的工作进展和感想 | git commit hash、代码 diff |
| 试剑发现的问题和建议 | Lint 报告、Lighthouse 原始分数 |
| 鸣锣的部署确认 | Vercel 部署日志、SSL 证书配置 |
| 局中人之间的互动和情绪 | Agent 的 prompt、tool 调用链 |

### 对话流的消息类型

| 类型 | 视觉 | 示例 |
|---|---|---|
| 发言 | 角色气泡（带头像和名字） | 🗣 说客："留存率只有 22%，说明现有产品不行" |
| 思考 | 淡色斜体气泡 | 🤔 诤臣：*在翻阅失败案例，准备反击...* |
| 行动 | 系统消息样式（居中小字） | 🎒 游商提交了采风报告，综合评分 72 分 |
| 裁决 | 特殊样式（印章 + 加粗） | ⚖️ 掌秤裁决：有条件地造。附带三个条件。 |
| 阶段转换 | 分隔线 + 章节标题 | ── 过堂结束，进入绘图 ── |
| 情绪 | 角色气泡尾部小字 | 🗣 说客（得意）/ ⚔️ 诤臣（不甘） |

### 对话流的数据来源

对话流不是一个新的数据表，而是从现有数据**组装**出来的：

| 对话流内容 | 数据来源 | 转换规则 |
|---|---|---|
| 游商的采风观察 | agent_logs（type: thought, visibility: public） | 直接展示 content |
| 采风报告摘要 | phases.output（scout 阶段） | 提取评分和各维度摘要，不展示原始 JSON |
| 说客/诤臣的辩论 | debates 表 | 直接展示 content（Markdown） |
| 局势分析 | 编排层生成，存入 events.detail | 展示局势条和转折点描述 |
| 掌秤裁决 | phases.output（council 阶段） | 提取 reason 和 conditions |
| 画师的设计思路 | agent_logs（type: thought） | 直接展示 |
| 蓝图摘要 | phases.output（architect 阶段） | 提取任务数和核心设计决策，不展示完整 JSON |
| 匠人的工作感想 | agent_logs（type: thought） | 直接展示 |
| 任务完成记录 | events（type: task_completed） | 展示任务标题，不展示代码细节 |
| 试剑的试剑结果 | phases.output（inspector 阶段） | 提取问题列表的描述和建议，不展示文件路径和行号 |
| 鸣锣确认 | phases.output（deployer 阶段） | 提取部署状态，不展示部署日志 |
| 局中人感想 | agent_logs（type: thought, 阶段完成后） | 直接展示 |

### 内容过滤规则

编排层在写入 agent_logs 时，用 `visibility` 字段控制哪些内容对外展示：

- `public`：展示在对话流和造物志中（局中人的观点、感想、互动）
- `internal`：仅内部可见（tool 调用细节、JSON 输出、技术日志）

Agent 在 SOUL.md 的行为准则中被要求：用 writeLog 记录思考时，用**角色的语言**而不是技术语言。

```
✅ writeLog({ type: "thought", visibility: "public",
     content: "这个赛道比我想的卷，但留存率数据很有意思。" })

❌ writeLog({ type: "thought", visibility: "public",
     content: "web_search 返回了 47 条结果，其中 12 条相关度 > 0.8" })
```

### 实时更新

对话流支持 SSE 实时推送。如果访客正在看一个进行中的造物令的对话流，新的消息会实时追加到底部，像聊天窗口一样。

```
SSE 事件 → 新的 debate_speech / agent_log / task_completed
  │
  └── 前端追加新的对话气泡，带 fadeIn 动画
      └── 如果是过堂发言，同时更新局势条
```
