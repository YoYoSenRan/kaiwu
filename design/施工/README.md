# 施工总览

## 模块依赖图

```
00-数据库
  │
  ├──→ 01-OpenClaw集成（需要 agents 表的 seed 数据）
  │
  ├──→ 02-API层（需要数据库表定义）
  │       │
  │       └──→ 03-编排层（需要 API 层的读写接口）
  │               │
  │               ├──→ 06-游商采风（需要编排层调度）
  │               │       │
  │               │       └──→ 07-过堂辩论（需要采风完成后触发）
  │               │               │
  │               │               └──→ 10-绘图锻造试剑（需要过堂通过后触发）
  │               │                       │
  │               │                       └──→ 11-鸣锣部署（需要试剑通过后触发）
  │               │
  │               └──→ 09-记忆系统（需要编排层的提炼钩子）
  │
  ├──→ 04-展示网站骨架（需要数据库查询）
  │       │
  │       └──→ 05-物帖提交（需要网站骨架 + 数据库）
  │               │
  │               └──→ 08-造物志（需要造物令数据积累）
  │
  └──→ 12-打磨（所有模块完成后）
```

## 执行顺序

| 序号 | 模块 | 依赖 | 预估工时 |
|---|---|---|---|
| 00 | 数据库 | 无 | 1 天 |
| 01 | OpenClaw 集成 | 00 | 1 天 |
| 02 | API 层 | 00 | 1-2 天 |
| 03 | 编排层 | 02 | 2-3 天 |
| 04 | 展示网站骨架 | 00 | 2 天 |
| 05 | 物帖提交 | 00 + 04 | 1-2 天 |
| 06 | 游商采风 | 01 + 03 | 2-3 天 |
| 07 | 过堂辩论 | 06 | 3-4 天 |
| 08 | 造物志 | 04 + 07 | 2-3 天 |
| 09 | 记忆系统 | 03 | 2 天 |
| 10 | 绘图锻造试剑 | 07 | 4-5 天 |
| 11 | 鸣锣部署 | 10 | 2 天 |
| 12 | 打磨 | 全部 | 持续 |

**可并行的模块**：
- 00 完成后，01 / 02 / 04 可以并行
- 04 + 05 可以和 01 + 02 + 03 并行
- 08 可以和 09 并行

## 每个模块的施工文档

每个模块目录下的 README.md 包含：
- **目标**：这个模块做完后能干什么
- **文件清单**：要创建/修改哪些文件
- **接口定义**：对外暴露什么、依赖什么
- **实现步骤**：按顺序列出具体步骤
- **验收标准**：怎么算做完了
- **参考文档**：对应的设计文档位置
