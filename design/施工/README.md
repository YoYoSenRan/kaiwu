# 施工总览

## 模块依赖图

```
00-项目初始化
  │
  ├──→ 01-数据库（需要 packages/db 骨架）
  │       │
  │       ├──→ 02-OpenClaw集成（需要 agents 表 seed 数据）
  │       │
  │       ├──→ 03-API层（需要数据库表定义）
  │       │       │
  │       │       └──→ 04-编排层（需要 API 层的读写接口）
  │       │               │
  │       │               ├──→ 07-游商采风（需要编排层调度）
  │       │               │       │
  │       │               │       └──→ 08-过堂辩论（需要采风完成后触发）
  │       │               │               │
  │       │               │               └──→ 11-绘图锻造试剑（需要过堂通过后触发）
  │       │               │                       │
  │       │               │                       └──→ 12-鸣锣部署（需要试剑通过后触发）
  │       │               │
  │       │               └──→ 10-记忆系统（需要编排层的提炼钩子）
  │       │
  │       └──→ 05-展示网站骨架（需要数据库查询）
  │               │
  │               └──→ 06-物帖提交（需要网站骨架 + 数据库）
  │                       │
  │                       └──→ 09-造物志（需要造物令数据积累）
  │
  └──→ 13-打磨（所有模块完成后）
```

## 执行顺序

| 序号 | 模块 | 依赖 | 预估工时 |
|---|---|---|---|
| 00 | 项目初始化 | 无 | 半天 |
| 01 | 数据库 | 00 | 1 天 |
| 02 | OpenClaw 集成 | 01 | 1 天 |
| 03 | API 层 | 01 | 1-2 天 |
| 04 | 编排层 | 03 | 2-3 天 |
| 05 | 展示网站骨架 | 01 | 2 天 |
| 06 | 物帖提交 | 01 + 05 | 1-2 天 |
| 07 | 游商采风 | 02 + 04 | 2-3 天 |
| 08 | 过堂辩论 | 07 | 3-4 天 |
| 09 | 造物志 | 05 + 08 | 2-3 天 |
| 10 | 记忆系统 | 04 | 2 天 |
| 11 | 绘图锻造试剑 | 08 | 4-5 天 |
| 12 | 鸣锣部署 | 11 | 2 天 |
| 13 | 打磨 | 全部 | 持续 |

**可并行的模块**：
- 01 完成后，02 / 03 / 05 可以并行
- 05 + 06 可以和 02 + 03 + 04 并行
- 09 可以和 10 并行

## 每个模块的施工文档

每个模块目录下的 README.md 包含：
- **目标**：这个模块做完后能干什么
- **文件清单**：要创建/修改哪些文件
- **接口定义**：对外暴露什么、依赖什么
- **实现步骤**：按顺序列出具体步骤
- **验收标准**：怎么算做完了
- **参考文档**：对应的设计文档位置
